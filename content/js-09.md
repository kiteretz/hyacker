## 回答

`try/catch/finally` でエラーを捕捉し、適切な処理を行います。

```js
try {
  // エラーが発生する可能性のある処理
  const data = JSON.parse(invalidJson);
} catch (error) {
  // エラー処理
  console.error('エラーが発生しました:', error.message);
} finally {
  // エラーの有無にかかわらず実行される
  console.log('処理完了');
}
```

## 解説

### Error オブジェクト

`error` オブジェクトのプロパティ：
- `error.message` : エラーメッセージ
- `error.name` : エラーの種類（`TypeError`、`RangeError` など）
- `error.stack` : スタックトレース

### エラーの種類で処理を分ける

```js
try {
  // 処理
} catch (error) {
  if (error instanceof TypeError) {
    console.error('型のエラー:', error.message);
  } else if (error instanceof RangeError) {
    console.error('範囲のエラー:', error.message);
  } else {
    throw error; // 想定外のエラーは再スロー
  }
}
```

### カスタムエラー

```js
class ValidationError extends Error {
  constructor(field, message) {
    super(message);
    this.name = 'ValidationError';
    this.field = field;
  }
}

function validateAge(age) {
  if (age < 0 || age > 150) {
    throw new ValidationError('age', '年齢は0〜150の範囲で入力してください');
  }
}
```

### 非同期処理のエラーハンドリング

```js
async function fetchData() {
  try {
    const response = await fetch('/api/data');
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    return await response.json();
  } catch (error) {
    console.error('データ取得エラー:', error);
    return null; // フォールバック値
  }
}
```
