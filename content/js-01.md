## 回答

現代の JavaScript では `var` は使わず、再代入が必要なら `let`、不要なら `const` を使います。

```js
// const: 再代入不可（オブジェクトの中身は変更可能）
const name = 'とも';
const user = { name: 'とも', age: 25 };
user.age = 26; // これは OK

// let: 再代入可能
let count = 0;
count = 1; // OK

// var: 使わない（スコープの問題があるため）
var old = 'avoid';
```

## 解説

### スコープの違い

`var` は関数スコープ、`let` と `const` はブロックスコープです。

```js
if (true) {
  var a = 1;   // if ブロックの外からアクセス可能
  let b = 2;   // if ブロックの外からアクセス不可
  const c = 3; // if ブロックの外からアクセス不可
}
console.log(a); // 1
console.log(b); // ReferenceError
```

### ホイスティング

`var` は宣言が関数の先頭に巻き上げられます（ホイスティング）。`let` と `const` もホイスティングされますが、初期化前にアクセスするとエラーになります（一時的デッドゾーン）。

### 推奨ルール

1. デフォルトは `const` を使う
2. 再代入が必要なときだけ `let` を使う
3. `var` は使わない
